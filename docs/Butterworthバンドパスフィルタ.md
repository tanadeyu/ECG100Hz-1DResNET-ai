# ButterworthバンドパスフィルタとFFTフィルタリングの比較

## 比較表

| 項目 | Butterworthバンドパスフィルタ (scipyのbutter + filtfilt) | FFTを使ったフィルタリング (FFT → 周波数カット → IFFT) |
|------|----------------------------------------------------------|--------------------------------------------------------|
| **処理の仕方** | 時間領域で直接信号を滑らかに通す（IIRフィルタ） | 信号を周波数領域に変換 → 不要周波数をゼロ → 逆変換 |
| **計算速度** | 速い（特に短いECGデータで線形時間） | 遅い（特に長い信号やバッチ処理でFFT/IFFTが重い） |
| **ECGへの適性** | 非常に良い（filtfiltでゼロ位相 → 波形歪み最小、QRS/P/T波を崩しにくい） | 急峻なカットオフでGibbs現象（リンギング）が発生しやすく、ECGの鋭いピークが歪む可能性大 |
| **位相歪み** | filtfiltを使えばゼロ（前後方向フィルタ） | 理想的にはゼロだが、窓関数なしだとエッジ効果・歪みが出やすい |
| **エッジ効果** | ほとんどなし（ゼロ位相） | 信号端で円周畳み込みによるアーティファクトが出やすい（ゼロパディング必須） |
| **実装の簡単さ** | 簡単（数行でOK） | 少し面倒（窓関数、ゼロパディング、オーバーラップ処理が必要な場合） |
| **ECG論文での使用頻度** | 圧倒的に多い（PTB-XLなど標準前処理） | ノッチフィルタ（50Hz除去）やスペクトル解析時に一部使うが、バンドパス本体では少ない |

## 結論（100Hz ECG、シンプルに済ませたいケース）

**Butterworthバンドパス（0.5〜40Hz + filtfilt）**は時間も短く、ECG波形をきれいに保てる → まさに「簡単で効果的」な選択。

FFT方式は計算が重く、ECG特有のリンギング（人工的な振動）が起きやすいため、避けた方が無難です。

## 推奨コード例

```python
from scipy.signal import butter, filtfilt

def butter_bandpass(data, lowcut=0.5, highcut=40.0, fs=100.0, order=4):
    nyq = 0.5 * fs
    low = lowcut / nyq
    high = highcut / nyq
    b, a = butter(order, [low, high], btype='band')
    y = filtfilt(b, a, data)
    return y
```

FFTはスペクトルを見る・高度な周波数編集が必要なときだけ使えば十分ですよ。

## 検証結果（2026-03-21）

**Butterworthバンドパスフィルタ（0.5-40Hz）の検証を実施**

- **データセット**: PTB-XL (21,837件, 100Hz)
- **モデル**: **1D-ResNet** (873万パラメータ)
- **比較**: BPFなし vs BPFあり

### 結果

| 指標 | BPFなし | BPFあり | 変化 |
|------|---------|---------|------|
| F1 (macro) | 0.6885 | 0.6933 | +0.0048 (+0.7%) |
| F1 (micro) | 0.7364 | 0.7375 | +0.0011 (+0.1%) |

### 結論

**効果は限定的（約0.5%向上）であり、採用見送り**

- PTB-XLデータセットは既に高品質でノイズが少ない
- 100Hzサンプリングでは0.5-40Hzのフィルタ効果が限定的
- モデルが十分に表現力が高く、前処理の依存度が低い

バンドパスフィルタ自体は有用だが、このタスクでは劇的な改善は見られなかった。
